# Miru
## 🔎 Overview 
**Miru** ('to watch' in japanese) is a personal movie and series library to track what you’ve watched and what you plan to watch. I'm starting this as a personal university project but who knows where it will end up... 👀

## 📈 Context & motivation
The rapid growth of streaming platforms and the constant release of new films and TV series have significantly increased the amount of audiovisual content available to viewers. As a result, users often struggle to remember which films or series they have already watched, which ones they enjoyed, and which titles they intended to watch later. This information is frequently scattered across different platforms or stored informally (notes, screenshots, recommendations from friends), making it difficult to manage in a clear and structured way.

While existing solutions and platforms may partially address this issue, they are often limited by platform dependency, lack of personalization, or restricted control over user data. Some tools focus primarily on recommendations rather than personal tracking, while others do not offer a simple and unified way to manage both watched content and watchlists.

The goal of this project is to design and develop a web-based application that allows users to create and manage their own personal library of films and TV series. Each user will be able to:

- Maintain a list of watched films and series
- Indicate whether they liked or disliked specific content
- Manage a watchlist of films and series they plan to watch

The application is designed as a multi-user platform, with individual accounts ensuring that each user’s library and preferences remain personal and independent. An administrative role is also planned to manage global aspects of the system if needed.

From a technical standpoint, this project serves as a practical application of key concepts studied during the course, including database modeling (MCD/MLD), user authentication and authorization, and web application architecture. Particular attention is given to data organization, scalability, and maintainability, in order to ensure that the system can evolve over time.

Finally, the project is designed with future extensibility in mind. Later versions may include integration with external movie and series databases (such as IMDb or similar APIs) to automatically retrieve content information, reduce manual data entry, and improve the overall user experience.

## 🎯 Scope and limitations
This project focuses on providing a personal content-tracking tool rather than a full-fledged streaming or recommendation platform. Its primary purpose is to allow users to organize and manage their own lists of films and TV series.

As such, the application does not aim to:

- Stream or host any video content
- Replace existing streaming platforms or commercial media databases
- Provide advanced recommendation algorithms based on artificial intelligence
- Act as a social network (no public profiles, comments, or social interactions are planned)
- Guarantee exhaustive or perfectly up-to-date media data in its initial versions

The project deliberately limits its scope in order to prioritize clarity, reliability, and maintainability. Features are selected based on educational value and technical feasibility within the given timeframe.

## 🔮 Future improvements
Although the initial version of the project focuses on core functionality, it is designed to be extensible and open to future enhancements.

Potential improvements include:

- Integration with external APIs (e.g., IMDb or similar services) to automatically retrieve film and series metadata
